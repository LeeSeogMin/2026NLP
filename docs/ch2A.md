# 1막. 뇌는 어떻게 판단하나 — 뉴런

<aside>
💡

**핵심 질문**: 우리 뇌는 수십억 개의 뉴런으로 이루어져 있다. 뉴런 하나가 하는 일은 놀랍도록 단순하다. 그런데 그 단순한 것들이 모이면 어떻게 "생각"이 될까?

</aside>

## 뉴런이 하는 일은 딱 하나

인간의 뇌에는 약 **1000억 개의 뉴런**(neuron)이 있다. 숫자는 어마어마하지만, 뉴런 하나가 하는 일은 의외로 단순하다.

> **받고 → 합치고 → 넘기거나 말거나**
> 

이게 전부다. 회사에서 일하는 직원 한 명을 떠올려보자.

|  | **뇌의 뉴런** | **회사의 직원** |
| --- | --- | --- |
| **받기** | 수상돌기(Dendrite)로 여러 뉴런에서 신호 수집 | 여러 부서에서 보고서를 받는다 |
| **합치기** | 세포체(Soma)에서 신호를 합산·처리 | 보고서를 종합해 "보고할 만한가?" 판단 |
| **넘기기** | 축삭돌기(Axon)로 다음 뉴런에 전달 | 판단 결과를 다음 사람에게 전달 |
| **성장** | 시냅스 가소성 — 경험에 따라 연결 강도 변화 | 경험이 쌓이면 "어떤 보고서가 중요한지" 감이 생긴다 |

![Untitled](https://prod-files-secure.s3.us-west-2.amazonaws.com/3835dc70-4e17-421a-b58e-c8cfd1017381/1dcb35a0-f144-4285-ac9b-ee80a39e3f0a/Untitled.png)

뉴런은 여러 입력 신호를 받아 **가중치를 적용해 합산**한다. 이 합이 **임계값을 넘으면 활성화**되어 신호를 출력하고, 넘지 않으면 침묵한다. 학습이란 결국 시냅스의 연결 강도(가중치)가 경험에 따라 변하는 것이다.

<aside>
✅

**기억할 것**: 뉴런 하나는 단순하다. 하지만 1000억 개가 연결되면 생각, 감정, 창의력이 탄생한다. 딥러닝은 이 원리를 빌려온 것이다.

</aside>

# 2막. 이것을 컴퓨터로 — 퍼셉트론

<aside>
💡

**핵심 질문**: 뉴런의 "받고 → 합치고 → 넘기거나 말거나"를 수학으로 쓸 수 있을까?

</aside>

## 퍼셉트론 = 뉴런의 수학 버전

1957년, 프랑크 로젠블라트(Frank Rosenblatt)가 **퍼셉트론**(Perceptron)을 만들었다. 뉴런이 하는 일을 수식 하나로 적은 것이다.

```
보고서(x) × 중요도(w) → 합산(Σ) + 기준선(b) → 판단(f) → 결과(y)

y = f(w₁x₁ + w₂x₂ + ... + wₙxₙ + b)
```

![image.png](attachment:ef7a471c-64fc-4c4c-b516-c6eb3f0dae38:image.png)

## 구성요소를 하나씩 뜯어보자

| **구성요소** | **수학 기호** | **회사 비유** | **역할** |
| --- | --- | --- | --- |
| 입력 | x₁, x₂, ... | 받은 보고서들 | 데이터의 특징(feature) |
| 가중치 | w₁, w₂, ... | "이 보고서가 얼마나 중요한지" | 입력의 중요도 (학습으로 조정) |
| 편향 | b | "보고 성향이 적극적인가 소극적인가" | 활성화 임계값 조절 |
| 활성화 함수 | f | "넘기냐 말냐"의 판단 기준 | 비선형 변환 |
| 출력 | y | 최종 보고 결과 | 0 또는 1 (이진 분류) |

## 퍼셉트론은 어떻게 학습하나?

처음에 이 직원은 **아무것도 모르는 신입**이다. 가중치는 아주 작은 무작위 값으로 시작한다.

1. **예측**: 입력을 받아 현재 가중치로 판단한다
2. **피드백**: 상사가 "그 판단 틀렸어"라고 알려준다 (오차 계산)
3. **조정**: 가중치를 살짝 수정한다 — 틀린 방향의 반대로
4. **반복**: 다음 데이터로 1~3을 반복

<aside>
📌

**구체적 예시**: 가중치 0.3으로 시작 → 입력 0.6 → 결과 0.18 → 정답은 1인데 0이라고 예측 → 오차 발생 → 가중치를 0.3 + (0.6 × 1 × 0.01) = 0.306으로 조정. 이런 미세한 조정이 수천 번 반복되면 점점 정확해진다.

</aside>

XOR 게이트: 입력값 두 개가 서로 다른 값을 갖고 있을때에만 출력값이 1이 되고, 입력값 두 개가 서로 같은 값을 가지면 출력값이 0

   -  이것은 종이를 휘어서 선을 그리는 것에 비유할 수 있으며, 이러한 매핑은 퍼셉트론의 은닉층에서 수행됨

   -  각 은닉층의 뉴런은 입력 데이터를 고차원 특징 공간으로 변환하여 원래의 비선형 문제를 선형 문제로 변환시키며, 다층 퍼셉트론은 XOR 문제를 포함한 다양한 비선형 문제를 해결할 수 있음

# 3막. 팀을 만들자 — 다층 퍼셉트론과 비선형성

<aside>
💡

**핵심 질문**: 직선 하나로 안 되면? 여러 직선을 조합하면 곡선이 될 수 있을까?

</aside>

## 조직을 만들다: 다층 퍼셉트론(MLP)

한 명의 직원(퍼셉트론)이 직선 하나만 그을 수 있다면, **여러 명을 조직으로 묶으면** 어떨까?

```
현장 수집팀 (입력층)        → 원시 데이터를 받아들인다
    ↓
중간 분석팀 (은닉층)        → 각자 다른 관점에서 분석한다
    ↓
최종 의사결정자 (출력층)    → 종합 판단을 내린다
```

![image.png](attachment:f4f5e70d-72fa-4c6f-b6d3-8898b558f997:image.png)

![image.png](attachment:f66f143a-0449-426c-8a47-8a28da9a89d2:image.png)

## 비선형성의 마법

은닉층의 각 노드는 **서로 다른 관점**으로 데이터를 바라본다.

- A 과장: 매출 관점에서 직선 하나
- B 과장: 고객 만족도 관점에서 직선 하나
- C 과장: 비용 관점에서 직선 하나

임원(출력층)이 이 **세 직선의 결과를 조합**하면 → **곡선**이 만들어진다.

<aside>
⭐

**직선 + 직선 + 직선을 잘 조합하면 어떤 곡선이든 근사할 수 있다.**

이것이 다층 퍼셉트론의 핵심 아이디어이자, 딥러닝이 강력한 이유다.

</aside>

## 깊이의 힘: 층이 깊어지면

| **층 깊이** | **이미지 인식 예시** | **자연어 처리 예시** |
| --- | --- | --- |
| 1층 (얕은 층) | 윤곽선, 색상, 텍스처 | 문법적 구조, 구문 관계 |
| 2층 (중간 층) | 눈, 코, 입 같은 부분 구조 | 문맥적 의미, 단어 간 참조 |
| 3층 (깊은 층) | 얼굴, 물체 같은 전체 개념 | 복잡한 추론, 추상적 개념 |

# 4막. 판단의 기준들 — 활성화 함수

<aside>
💡

**핵심 질문**: 뉴런이 "넘길지 말지" 판단하는 기준, 즉 활성화 함수에는 어떤 종류가 있을까?

</aside>

직원마다 **성격이 다르듯**, 활성화 함수도 종류가 다르다.

## 주요 활성화 함수

| **함수** | **성격 비유** | **출력 범위** | **특징** |
| --- | --- | --- | --- |
| **계단 함수** | 흑백논리형 — "된다/안 된다" | 0 또는 1 | 퍼셉트론의 원조. 미분 불가 |
| **시그모이드** | 신중형 — "70%쯤 될 것 같다" | 0 ~ 1 | 부드러운 곡선. 양 끝에서 반응 둔화(기울기 소실) |
| **ReLU** | 실용형 — "음수면 무시, 양수면 그대로" | 0 ~ ∞ | 단순·빠름. 현재 가장 많이 사용 |
| **Leaky ReLU** | 개선형 — "음수도 아주 살짝 반영" | -∞ ~ ∞ | ReLU의 "죽은 뉴런" 문제 해결 |
| **소프트맥스** | 투표형 — "전체 중에서 각자 몇 %?" | 0 ~ 1 (합=1) | 다중 분류 출력층 전용 |

![](https://miro.medium.com/max/720/1*XxxiA0jJvPrHEJHD4z893g.png)

## 선택 가이드 (실전)

<aside>
📌

- **은닉층** → ReLU (기본값. 대부분의 경우 이것으로 충분)
- **이진 분류 출력층** → 시그모이드 ("맞다/아니다" 확률)
- **다중 분류 출력층** → 소프트맥스 ("A일 확률 30%, B일 확률 70%")

# 5막. 얼마나 틀렸나 — 손실 함수

<aside>
💡

**핵심 질문**: 조직(신경망)이 만들어졌다. 그런데 **"우리가 얼마나 틀렸는지"** 측정할 수 없다면 개선도 없다. 어떻게 측정할까?

</aside>

손실 함수(Loss Function)는 **성과 평가 지표**다. 모델의 예측값과 정답 사이의 차이를 **숫자 하나로** 요약해준다. 이 숫자가 작을수록 좋은 모델이다.

## 1. 평균 제곱 오차 (MSE) — 회귀 문제용

```
MSE = (1/n) × Σ(정답 - 예측)²
```

- 예측값과 정답의 **거리의 제곱** 평균
- 많이 틀릴수록 **제곱으로 벌점**이 커진다
- 주로 숫자를 예측하는 **회귀 문제**에 사용

## 2. 교차 엔트로피 (Cross-Entropy) — 분류 문제용

- "정답일 확률을 얼마나 높게 줬는가"를 측정
- **확신 있게 틀리면 벌점이 폭증**한다

## 왜 분류에서 MSE 대신 교차 엔트로피인가?

|  | **MSE** | **교차 엔트로피** |
| --- | --- | --- |
| 틀렸을 때 벌점 증가 | 완만하게 증가 | 급격하게 폭증 |
| 학습 신호 | 약한 신호 → 학습 느림 | 강한 신호 → 학습 빠름 |
| 분류 적합도 | 회귀에 적합 | 분류에 최적화 |

![실제값에 가까워질수록, 즉 1에 가까울수록 두 함수는 낮은 값을 보이지만, 실제값에서 멀어질수록 크롤스 엔트로피 점수는 매우 커진다. ](https://prod-files-secure.s3.us-west-2.amazonaws.com/3835dc70-4e17-421a-b58e-c8cfd1017381/229a16e7-8c6b-477d-b792-7492c567c226/Untitled.png)

실제값에 가까워질수록, 즉 1에 가까울수록 두 함수는 낮은 값을 보이지만, 실제값에서 멀어질수록 크롤스 엔트로피 점수는 매우 커진다. 

그래프에서 볼 수 있듯이, 예측이 실제값에서 멀어질수록 크로스 엔트로피 손실은 MSE보다 더 급격히 증가한다. 이는 잘못된 예측에 더 강력한 학습 신호를 제공한다.

# 6막. 어느 방향으로 고칠까 — 경사하강법

## 안개 낀 산에서 내려가기

> 안개 낀 산 꼭대기에서 눈을 감고 내려가야 한다.
할 수 있는 건 **발밑의 기울기(경사)만 느끼는 것**.
가장 가파르게 내려가는 방향으로 한 걸음씩 내딛는다.
이것이 **경사하강법(Gradient Descent)**이다.
> 

## 수식 한 줄

```
w_new = w - η × ∂E/∂w

새 가중치 = 현재 가중치 - 학습률 × 기울기(그래디언트)
```

| **개념** | **수학** | **산 비유** |
| --- | --- | --- |
| 그래디언트 ∂E/∂w | 손실 함수의 가중치에 대한 편미분 | 발밑의 기울기 — 어느 방향이 내리막인지 |
| 학습률 η | 업데이트 크기 조절 상수 | 보폭 — 너무 크면 지나치고, 너무 작으면 느리다 |
| 전역 최소값 | 손실이 가장 작은 지점 | 산의 가장 낮은 골짜기 — 우리의 목표 |

## 학습률이 왜 중요한가

- **너무 큰 학습률**: 골짜기를 지나쳐서 발산
- **너무 작은 학습률**: 내려가는 데 시간이 너무 오래 걸림
- **적절한 학습률**: 안정적으로 최적점에 도달

## 세 가지 방식: 시험 공부에 비유하면

| **방식** | **비유** | **장점** | **단점** |
| --- | --- | --- | --- |
| **배치 (Batch)** | 시험 범위 전체를 다 복습한 후 방향 수정 | 정확한 방향 | 시간이 너무 오래 걸림 |
| **SGD** | 랜덤 1페이지만 보고 바로 수정 | 빠른 피드백 | 방향이 불안정 |
| **미니배치** ⭐ | 한 챕터씩 보고 수정 — **실전 표준** | 속도와 안정성의 균형 | 배치 크기 선택 필요 |

## 고급 최적화: 모멘텀과 Adam

기본 경사하강법에는 한계가 있다: 평탄한 곳에서 느려지고, 지역 최소값에 갇힐 수 있다.

### 모멘텀 (Momentum)

> 매끄러운 표면을 굴러 내려오는 공을 상상하자. 공은 **관성**이 있어서 작은 언덕도 넘어갈 수 있고, 같은 방향이 계속되면 점점 빨라진다.
> 

### Adam (Adaptive Moment Estimation) ⭐

> 특별한 등산객을 상상하자. 이 등산객은 세 가지 능력이 있다:
1. **방향 기억** — 최근 어디로 움직여왔는지 (모멘텀)
2. **지형 적응** — 굴곡이 심한 곳은 작은 걸음, 평탄한 곳은 큰 걸음
3. **초기 신중함** — 처음엔 정보가 부족하니 더 조심스럽게
> 

Adam은 현대 딥러닝의 **기본 옵티마이저**다. 대부분의 문제에서 기본 설정으로도 잘 동작한다.

## 레스토랑 주방 이야기

레스토랑 주방에 여러 팀(층)이 있다고 상상하자.

### 순전파: 주문에서 요리까지

```
주문 (입력)  →  재료팀 (입력층)  →  1차 조리 (은닉층1)  →  2차 조리 (은닉층2)  →  플레이팅 (출력층)  →  손님
```

"스테이크 미디엄으로 하나" → 재료 준비 → 굽기 → 양념 → 플레이팅 → 손님에게 전달

### 역전파: 불만이 거꾸로 전달된다

```
손님: "너무 익었어요" (오차)
         ↓
플레이팅팀 ← "조리에 문제가 있나 봐요"
         ↓
2차 조리팀 ← "양념 시간이 길어서 더 익은 것 같아요" → 양념 시간 줄이자
         ↓
1차 조리팀 ← "고기를 너무 오래 구웠나 봐요" → 굽기 시간 줄이자
         ↓
재료팀    ← "고기가 얇아서 빨리 익었어요" → 두껍게 자르자
```

<aside>
⭐

**역전파의 본질**: 최종 오차를 뒤에서 앞으로 전달하면서, **각 단계가 오차에 얼마나 기여했는지** 계산하고, 그만큼 조정한다.

</aside>

![역전파의 원리 — 레스토랑 주방 비유](https://files.catbox.moe/07egv0.png)

역전파의 원리 — 레스토랑 주방 비유

## 수학적 원리: 연쇄법칙 (Chain Rule)

역전파의 수학적 기초는 미적분의 **연쇄법칙**이다.

> 함수가 겹겹이 쌓여 있을 때, 전체 미분 = 각 단계 미분의 곱
> 

```
y = f(g(x)) 일 때

dy/dx = (dy/dg) × (dg/dx)
```

신경망에 적용하면:

```
∂E/∂w₁ = (∂E/∂y) × (∂y/∂h) × (∂h/∂w₁)

= 출력층 기여 × 은닉층 기여 × 입력층 기여
```

각 층의 기여도를 **뒤에서 앞으로 연쇄적으로 곱해가며** 모든 가중치의 그래디언트를 계산한다.

## 수치 예시: 직접 따라가 보자

- 📊 2층 신경망 역전파 계산 (클릭해서 펼치기)
    
    **네트워크 구성**
    
    - 입력: x₁ = 0.5, x₂ = 0.3
    - 가중치: w₁=0.7, w₂=0.3, w₃=0.4, w₄=0.6, w₅=0.55, w₆=0.45
    - 타겟: y = 1
    - 활성화 함수: 시그모이드
    
    **1단계: 순전파**
    
    - z₁ = 0.7×0.5 + 0.3×0.3 = 0.44 → h₁ = σ(0.44) ≈ 0.608
    - z₂ = 0.4×0.5 + 0.6×0.3 = 0.38 → h₂ = σ(0.38) ≈ 0.594
    - z₃ = 0.55×0.608 + 0.45×0.594 = 0.601 → o₁ = σ(0.601) ≈ 0.646
    - 오차: C = (1 - 0.646)² ≈ 0.125
    
    **2단계: 역전파 (출력층 → 은닉층)**
    
    - ∂C/∂o₁ = -2(1 - 0.646) ≈ -0.708
    - ∂o₁/∂z₃ = 0.646 × (1 - 0.646) ≈ 0.229
    - 출력층 오차 δ = -0.708 × 0.229 ≈ -0.162
    - ∂C/∂w₅ = δ × h₁ = -0.162 × 0.608 ≈ -0.098
    
    **3단계: 가중치 업데이트 (학습률 η = 0.1)**
    
    - w₅: 0.55 → 0.55 + 0.1 × 0.098 = **0.56** ✓ 오차 줄어드는 방향!
    - w₆: 0.45 → **0.46**, w₁: 0.70 → **0.701**
    
    한 사이클로 모든 가중치가 오차를 줄이는 방향으로 이동했다!
    

## 주의: 그래디언트 소실 문제

시그모이드의 미분값은 최대 0.25다. 층마다 이 값이 곱해지면?

```
10층이고 각 층의 미분값이 0.1이라면:
0.1¹⁰ = 0.0000000001

→ 입력층 근처에 도달할 때쯤 그래디언트가 거의 0!
→ 앞쪽 층은 거의 학습되지 않는다
```

**현대적 해결책:**

| **해결책** | **원리** |
| --- | --- |
| ReLU / Leaky ReLU | 시그모이드처럼 포화되지 않아 그래디언트가 잘 전달됨 |
| 배치 정규화 (Batch Norm) | 층마다 입력값을 정규화해 극단값을 줄임 |
| 잔차 연결 (ResNet) | 그래디언트가 직접 전달되는 지름길 추가 |
| Xavier/He 초기화 | 가중치를 너무 크거나 작지 않게 시작 |

## 현대적 구현: PyTorch에서는 4줄

```python
outputs = model(inputs)        # 순전파
loss = criterion(outputs, targets)  # 손실 계산

optimizer.zero_grad()          # 그래디언트 초기화
loss.backward()                # 역전파 (자동!)
optimizer.step()               # 가중치 업데이트
```
