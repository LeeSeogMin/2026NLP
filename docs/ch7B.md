### 문항 1-3. PyTorch Tensor 와 NumPy 배열
다음 PyTorch 코드를 실행했을 때 결과는?

```python
import torch
x = torch.tensor([1.0, 2.0, 3.0], requires_grad=True)
y = x ** 2
z = y.sum()
z.backward()
print(x.grad)
```

① tensor([0., 0., 0.])
② tensor([2., 4., 6.]) ✓
③ tensor([1., 2., 3.])
④ AttributeError (grad 속성 없음)

**해설**: `requires_grad=True`로 설정되면 Autograd가 활성화된다. z = (x²).sum() 에서 ∂z/∂x = 2x 이므로, 각 원소의 그래디언트는 [2×1, 2×2, 2×3] = [2, 4, 6]이다.

### 문항 1-4. Autograd의 의미
**자동 미분(Autograd)**의 핵심 장점은?

① 손으로 계산한 미분보다 정확하지 않다
② **역전파 과정을 자동으로 계산하여 개발자가 손수 그래디언트를 계산할 필요가 없다** ✓
③ 모든 연산 속도를 100배 높인다
④ GPU 메모리를 절약한다

**해설**: Autograd는 신경망의 각 레이어와 가중치에 대한 그래디언트(손실에 대한 편미분)를 자동으로 역추적하여 계산한다. 개발자는 forward pass만 정의하면 backward pass가 자동으로 생성된다.

## 2주차 범위: 신경망 구조 및 PyTorch 패턴 (8문항)

### 문항 2-1. 퍼셉트론의 한계
**XOR 문제**가 중요한 이유는?

① XOR은 일반적인 문제이고 해결하기 쉽다
② **은닉층이 없는 단층 퍼셉트론(선형 분류기)은 XOR 같은 비선형 문제를 풀 수 없다** ✓
③ XOR 연산은 신경망에서 작동하지 않는다
④ XOR 문제를 풀면 모든 딥러닝 문제가 해결된다

**해설**: XOR은 "평면 위의 한 직선으로는 두 클래스를 분리할 수 없는" 비선형 문제의 대표 사례다. 이는 은닉층(비선형 활성화)이 필요함을 입증했고, 이것이 다층 신경망(MLP)의 동기가 되었다.

---

### 문항 2-2. 활성화 함수의 역할
다음 중 **활성화 함수가 필요한 이유**는?

① 계산 속도를 높이기 위해
② **비선형성을 도입하여 복잡한 패턴을 학습할 수 있게 하기 위해** ✓
③ 메모리 사용량을 줄이기 위해
④ GPU 메모리 캐싱을 개선하기 위해

**해설**: 활성화 함수가 없으면 아무리 깊은 신경망도 결국 선형 변환의 연쇄일 뿐이다 (선형×선형=선형). 활성화 함수(ReLU, GELU, Sigmoid 등)의 비선형성이 신경망이 곡선, 비선형 결정 경계를 표현하도록 한다.

### 문항 2-3. Cross-Entropy 손실 함수
다음은 분류 문제에서 Cross-Entropy 손실을 사용하는 이유를 설명한 것이다. 가장 적절한 설명은?

① MSE(평균 제곱 오차)와 동일한 결과를 준다
② **확률 분포의 불일치를 측정하며, 분류 문제에 특화된 손실 함수이다** ✓
③ 회귀 문제에만 사용된다
④ 손실 값이 항상 0~1 범위로 제한된다

**해설**: Cross-Entropy는 모델 예측 확률 분포와 실제 분포(원-핫 인코딩)의 차이를 KL 발산으로 측정한다. 분류 문제에서 정답에 높은 확률을 할당하도록 유도한다. MSE는 출력값이 연속인 회귀에 더 적합하다.

### 문항 2-4. 경사 하강법의 핵심
**경사 하강법(Gradient Descent)**을 가장 잘 설명한 비유는?

① 산 위에서 무작위로 돌아다니기
② **어두운 산에서 손전등 없이 내려오되, 발이 가장 가파르게 내려가는 방향을 따라 걷기** ✓
③ 산의 정상에 오르기 위해 항상 위쪽을 향해 가기
④ 산 전체 지도를 먼저 그린 뒤 최적 경로를 선택하기

**해설**: 경사 하강법은 현재 위치의 기울기(그래디언트)를 따라 손실을 줄이는 방향으로 가중치를 업데이트한다. 국소 최솟값(로컬 미니마)에 빠질 수 있지만, 신경망에서는 대부분 충분히 좋은 해를 찾는다.

### 문항 2-5. PyTorch 훈련 루프 구조
다음 훈련 루프 코드에서 **누락된 부분**은?

```python
model = MyMLP()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
for epoch in range(num_epochs):
    for batch_X, batch_y in train_loader:
        # ① ???
        output = model(batch_X)
        loss = criterion(output, batch_y)
        # ② ???
        loss.backward()
        # ③ ???
```

① optimizer.zero_grad(), optimizer.step()
② backward(), optimizer.step()
③ **optimizer.zero_grad() / loss.backward() / optimizer.step()** ✓
④ model.eval(), model.zero_grad()

**해설**: PyTorch 훈련 루프의 표준 패턴은:
1. `optimizer.zero_grad()`: 이전 배치의 그래디언트 초기화
2. `loss.backward()`: 역전파로 그래디언트 계산
3. `optimizer.step()`: 계산된 그래디언트로 가중치 업데이트

### 문항 2-6. 배치 정규화의 목적
**배치 정규화(Batch Normalization)**의 주요 효과는?

① 모델의 크기를 줄인다
② **학습을 안정화하고 더 높은 학습률을 사용할 수 있게 하며, 과적합을 감소시킨다** ✓
③ 훈련 시간을 절반으로 줄인다
④ 정확도를 항상 10% 이상 높인다

**해설**: 배치 정규화는 각 배치의 입력을 정규화(평균 0, 표준편차 1)하여 내부 공변량 이동(Internal Covariate Shift)을 줄인다. 결과적으로 학습이 안정적이고 빠르게 수렴한다.

### 문항 2-7. 과적합 방지: Dropout
**Dropout의 작동 원리**를 가장 정확히 설명한 것은?

① 입력 데이터를 무작위로 삭제한다
② **훈련 중 랜덤하게 뉴런의 활성화를 비활성화(0으로 설정)하여 공동 적응을 방지한다** ✓
③ 가중치를 무작위로 재초기화한다
④ 배치 정규화와 동일한 기능을 한다

**해설**: Dropout은 훈련 중 일부 뉴런을 임의로 "꺼서" 특정 뉴런에 과도하게 의존하는 것을 방지한다. 테스트 시에는 모든 뉴런이 활성화된다 (또는 가중치를 조정). 앙상블 효과를 만들어 일반화 성능을 높인다.

### 문항 2-8. 학습률의 영향
다음 세 가지 학습률로 모델을 훈련했을 때 **예상되는 결과**는?

- 학습률 0.1 (너무 크다)
- 학습률 0.001 (적절함)
- 학습률 0.00001 (너무 작다)

① 0.1과 0.00001 모두 빠르게 수렴한다
② **0.1은 발산하고, 0.00001은 매우 천천히 수렴하며, 0.001이 가장 빠르게 좋은 해에 도달한다** ✓
③ 학습률이 작을수록 항상 더 좋다
④ 학습률은 결과에 영향을 주지 않는다

**해설**: 학습률이 너무 크면 손실이 진동하거나 발산하고, 너무 작으면 수렴이 매우 느리다. 적절한 학습률을 선택하는 것은 딥러닝의 핵심 하이퍼파라미터 튜닝이다.

## 3주차 범위: 임베딩, RNN/LSTM, Attention (8문항)

### 문항 3-1. Word2Vec 원리
**Word2Vec (Skip-gram)**의 핵심 아이디어는?

① 단어를 알파벳 순서로 정렬한다
② **주변 단어를 보고 중심 단어를 예측하거나, 중심 단어로부터 주변 단어를 예측하는 과정에서 단어 임베딩을 학습한다** ✓
③ 문장 전체를 한 번에 하나의 벡터로 변환한다
④ 모든 단어를 동일한 거리에 배치한다

**해설**: Word2Vec은 "비슷한 문맥에 등장하는 단어는 비슷한 의미를 갖는다"는 가정 하에, 문맥(주변 단어) 예측 과제의 부산물로 임베딩을 학습한다. 결과적으로 "왕 - 남자 + 여자 ≈ 여왕" 같은 의미적 관계가 벡터 공간에서 나타난다.

---

### 문항 3-2. 임베딩의 의미
다음 중 **임베딩(Embedding)**의 역할을 가장 잘 설명한 것은?

① 이진 벡터(0/1)로 단어를 표현한다
② 단어의 길이에 따라 벡터 크기를 결정한다
③ **높은 차원의 원-핫 벡터를 저차원의 밀집 벡터로 변환하여, 의미적 유사도를 반영한다** ✓
④ 텍스트를 압축만 하고 의미 정보는 손실한다

**해설**: 임베딩은 원-핫 벡터(vocab_size 차원, 거의 모두 0)의 비효율성을 해결한다. 대신 학습 가능한 저차원 벡터(보통 50~300차원)로 표현하며, 이 벡터 공간에서 의미적으로 가까운 단어들이 인접하다.

---

### 문항 3-3. RNN의 구조와 장점
**RNN(순환 신경망)**의 핵심 특징은?

① 모든 입력을 동시에 처리한다
② **이전 시점의 숨겨진 상태를 다음 시점으로 전달하여 순차적 정보를 처리한다** ✓
③ 단어를 독립적으로 처리하고 순서 정보는 무시한다
④ 문장의 길이와 무관하게 항상 동일한 계산량을 사용한다

**해설**: RNN은 시간 축에 따라 순환하는 구조로, 현재 입력과 이전 숨겨진 상태(hₜ₋₁)를 받아 새로운 숨겨진 상태(hₜ)를 생성한다. 이를 통해 "저는 프랑스에 있었고... 나는 __를 말한다"처럼 문맥을 기억한다.

---

### 문항 3-4. 장기 의존성 문제
**RNN의 장기 의존성 문제(vanishing/exploding gradient)**를 설명한 것 중 맞는 것은?

① RNN은 장기 의존성 문제가 없다
② **역전파 과정에서 그래디언트가 지수적으로 감소(또는 증가)하여, 먼 과거의 정보가 현재에 거의 영향을 주지 못한다** ✓
③ 장기 의존성 문제는 Dropout으로 완벽히 해결된다
④ 모든 신경망에서 동일하게 나타난다

**해설**: RNN에서 역전파할 때, 타임스텝이 많을수록 (예: 100개 단어) 그래디언트를 계속 곱하게 된다. 활성화 함수의 미분값이 보통 0.1 미만이므로, 100번 곱하면 거의 0에 가까워진다(소실). 이는 긴 문맥에서 초반 정보의 영향이 무시됨을 의미한다.

---

### 문항 3-5. LSTM의 개선
**LSTM(Long Short-Term Memory)**이 RNN의 문제를 해결한 방법은?

① 더 깊은 신경망을 쌓는다
② **Cell State라는 컨베이어 벨트를 도입하고, Forget/Input/Output Gate로 정보 흐름을 선택적으로 제어한다** ✓
③ 모든 연산을 병렬화한다
④ 임베딩 크기를 증가시킨다

**해설**: LSTM은 Cell State (Cₜ)라는 별도의 정보 흐름을 추가하여 장기 정보를 보존한다. Forget Gate는 불필요한 정보를 버리고, Input Gate는 새 정보를 추가하며, Output Gate는 숨겨진 상태를 출력한다. 이를 통해 그래디언트가 좀 더 안정적으로 역전파된다.

---

### 문항 3-6. Attention 메커니즘
**Attention**의 핵심 동작을 설명한 것은?

① 모든 입력에 동일한 가중치를 할당한다
② **주어진 Query에 대해, 모든 Key 중에서 가장 관련 있는 것(높은 유사도)에 높은 가중치를 할당하고, 해당 Value들의 가중 평균을 계산한다** ✓
③ 순서대로 입력을 처리한다
④ 첫 입력에만 집중한다

**해설**: Attention은 "이 단어를 이해하기 위해 문장의 어떤 부분에 주목해야 하는가?"라는 질문에 답한다. 수학적으로는 softmax(QKᵀ/√dₖ) · V 형태로, Query와 Key의 유사도를 기반으로 Value의 가중 평균을 구한다.

---

### 문항 3-7. 코드로 이해하는 Attention
다음은 Scaled Dot-Product Attention의 의사 코드이다. 빈칸을 채우시오.

```
Q, K, V = queries, keys, values
scores = Q @ K.T / sqrt(d_k)    # 유사도 계산
weights = softmax(scores, dim=-1)  # 가중치 정규화
output = weights @ V  # 가중 평균
```

이 과정에서 `sqrt(d_k)`를 나누는 이유는?

① 계산 속도를 높이기 위해
② **벡터 차원이 클 때 스코어 값이 너무 커지는 것을 방지하여, softmax의 그래디언트를 안정화하기 위해** ✓
③ 메모리 사용량을 줄이기 위해
④ 항상 출력값을 0~1 범위로 제한하기 위해

**해설**: Attention Score는 내적(QKᵀ) 계산으로 얻어진다. 벡터 차원(dₖ)이 크면 내적값이 매우 커져서 softmax의 그래디언트가 0에 가까워진다(그래디언트 소실). √dₖ로 스케일링하면 이를 방지한다.

---

### 문항 3-8. Self-Attention vs Cross-Attention
다음 중 **Self-Attention**의 특징은?

① Query, Key, Value가 서로 다른 출처에서 온다
② **Query, Key, Value가 모두 동일한 입력(예: 같은 문장)에서 생성되며, 문장 내 단어들 간의 상호작용을 모델링한다** ✓
③ 두 개의 서로 다른 언어 간 관계를 모델링한다
④ 시간 순서대로 처리해야 한다

**해설**: Self-Attention은 "이 문장의 각 단어가 다른 단어들과 어떤 관계가 있는가?"를 포착한다. 반면 Cross-Attention은 인코더와 디코더처럼 두 개의 다른 입력 간 관계를 모델링한다 (예: 영어 문장과 그에 해당하는 프랑스어 문장).

---

## 4주차 범위: Transformer, Positional Encoding, Self-Attention (8문항)

### 문항 4-1. Transformer의 혁신
**Transformer**가 RNN/LSTM을 능가한 가장 중요한 이유는?

① 더 깊은 신경망을 사용한다
② **Self-Attention으로 모든 토큰 쌍의 상호작용을 병렬로 계산할 수 있어, 순차 처리(RNN)의 한계를 극복했다** ✓
③ 임베딩 차원이 크다
④ 더 많은 파라미터를 가진다

**해설**: RNN은 순차적으로만 처리 가능하다 (시간 스텝 t를 계산하려면 t-1을 먼저 완료해야 함). 반면 Transformer의 Self-Attention은 모든 위치 간의 상호작용을 동시에 계산할 수 있어, GPU의 병렬 처리를 최대로 활용한다. 이는 훈련 속도를 크게 높이고, 더 긴 시퀀스를 처리할 수 있게 한다.

---

### 문항 4-2. Positional Encoding
**Positional Encoding이 필요한 이유**는?

① 메모리 사용량을 줄이기 위해
② **Self-Attention은 위치 정보를 인식하지 못하므로 (모든 위치가 동등), 단어의 위치를 명시적으로 임베딩에 추가해야 한다** ✓
③ 임베딩 벡터의 크기를 증가시키기 위해
④ 계산 속도를 높이기 위해

**해설**: "나는 개를 본다"와 "개는 나를 본다"는 단어 순서가 다르지만, Self-Attention만 사용하면 두 경우가 동일하게 처리된다 (입력이 순환하지 않고, 주의 계산이 위치 불변적이므로). Positional Encoding은 위치 정보를 벡터로 인코딩하여, 모델이 단어 순서를 구분할 수 있게 한다.

---

### 문항 4-3. Positional Encoding의 형식
**Original Transformer의 Positional Encoding 수식**은?

PE(pos, 2i) = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))

이 수식의 특징으로 맞는 것은?

① 모든 위치에서 동일한 값을 갖는다
② **사인과 코사인의 조합으로 상대적 위치를 표현하며, 모델이 다양한 거리의 의존성을 학습할 수 있게 한다** ✓
③ 절대 위치를 1차 함수로 인코딩한다
④ 위치 정보가 없다

**해설**: 사인/코사인 함수는 주기성을 가지므로, 상대적 거리 관계를 표현할 수 있다. 또한 학습이 아닌 고정 함수이므로, 훈련 데이터보다 긴 시퀀스에도 일반화된다. 일부 최신 모델은 학습 가능한 위치 임베딩을 사용하기도 한다.

---

### 문항 4-4. Multi-Head Attention
**Multi-Head Attention의 목적**은?

① 계산 시간을 줄인다
② **여러 개의 Attention Head로 서로 다른 representation subspace에서 정보를 병렬로 학습하여, 다양한 관점의 관계를 포착한다** ✓
③ 단일 Head Attention과는 완전히 다른 원리를 사용한다
④ 메모리 사용량을 반으로 줄인다

**해설**: Multi-Head Attention은 n개의 독립적인 Attention Head를 병렬로 실행한다. 각 Head는 d_model/n 차원에서 작동하여, 서로 다른 의미 공간에서 패턴을 학습한다. 예를 들어 한 Head는 문법 관계, 다른 Head는 의미 관계를 학습할 수 있다. 최종적으로 모든 Head의 결과를 연결(concatenate)한다.

---

### 문항 4-5. Transformer 아키텍처
**Transformer Encoder-Decoder 구조**를 설명한 것 중 맞는 것은?

① Encoder와 Decoder는 동일한 구조이다
② **Encoder는 입력 시퀀스를 처리하는 Self-Attention 스택이고, Decoder는 Encoder의 출력과 생성 중인 시퀀스 간의 Cross-Attention으로 디코딩한다** ✓
③ Encoder는 필요 없고 Decoder만 사용한다
④ Encoder와 Decoder는 독립적으로 훈련된다

**해설**: Transformer Encoder는 Self-Attention과 Feed-Forward 레이어를 여러 번 반복하여, 입력의 문맥 표현을 학습한다. Decoder는 Encoder의 출력(Key, Value)과 자신의 이전 출력(Query)을 Cross-Attention으로 결합하여 다음 토큰을 생성한다. 예: 기계번역에서 Encoder는 영어, Decoder는 프랑스어를 처리한다.

---

### 문항 4-6. Feed-Forward 네트워크
Transformer의 각 Encoder/Decoder 레이어에는 Multi-Head Attention 외에 **Feed-Forward 네트워크**가 있다. 이 구조는?

① 시간 정보를 추가하는 모듈이다
② **두 개의 선형 레이어(선형 → ReLU → 선형)로, Attention의 결과를 non-locally 변환하고 표현력을 높인다** ✓
③ Attention과 동일한 기능을 한다
④ 메모리 최적화 모듈이다

**해설**: Transformer의 각 레이어는 (1) Multi-Head Attention + (2) Feed-Forward 네트워크를 순차적으로 적용한다. Feed-Forward는 위치별(position-wise)로 동일한 MLP를 적용하여, Attention이 포착한 관계를 바탕으로 각 위치의 표현을 변환한다. 보통 d_model → 4d_model → d_model 형태의 확장과 축소 구조를 가진다.

---

### 문항 4-7. 마스킹(Masking)
**Decoder의 Self-Attention에서 마스킹이 필요한 이유**는?

① 계산을 빠르게 하기 위해
② **미래 토큰(아직 생성되지 않은 토큰)을 현재 시점에서 "볼 수 없도록" 하여, 자동 회귀 생성(autoregressive generation)을 보장한다** ✓
③ 메모리 사용량을 줄이기 위해
④ 모델의 복잡도를 낮추기 위해

**해설**: Transformer Decoder는 이전에 생성한 토큰들만 보고 다음 토큰을 예측해야 한다 (자동 회귀). 만약 마스킹이 없으면, Attention이 미래 토큰을 참조할 수 있어서 (훈련/테스트 불일치) 모델이 제대로 학습되지 않는다. 마스킹(보통 -∞ 할당)으로 미래 위치의 Attention 가중치를 0으로 만든다.

---

### 문항 4-8. Attention 가중치 시각화
다음 문장 "나는 학교에 간다"에서 Attention이 계산한 가중치를 시각화하면, 어떤 패턴이 기대되는가?

나 | 는 | 학교 | 에 | 간다
---|---|---|---|---
[0.1, 0.05, 0.4, 0.35, 0.1] | [0.05, 0.2, 0.3, 0.35, 0.1] | ... | ... | ...

이 가중치는 무엇을 의미하는가?

① 각 단어가 다음 단어만 주목한다
② **"나"는 "학교"에 40%, "에"에 35%의 주의를 할당하며, 이는 "학교에"가 목적지 정보를 형성함을 모델이 인식했음을 의미한다** ✓
③ 모든 단어에 동일한 가중치를 할당했다
④ 가중치의 합이 1이 아니다

**해설**: Attention 가중치는 학습된 의존성 관계를 드러낸다. "나"가 "학교에"에 주의를 집중하는 것은 "학교에 간다"에서 문법적/의미적으로 중요한 구조를 모델이 포착했음을 의미한다. 이러한 Attention 시각화는 모델의 의사결정을 해석하는 데 유용하다.

---

## 5주차 범위: BERT, GPT, Transfer Learning (8문항)

### 문항 5-1. 사전학습(Pre-training)의 목표
**사전학습 언어 모델**의 핵심 목적은?

① 목표 작업(분류, 요약 등)을 직접 해결하기
② **대규모 비라벨 데이터에서 자동 목표(자기지도 학습)로 언어 표현을 학습하여, 다양한 하위 작업(downstream task)에 전이 가능한 일반적 지식을 획득한다** ✓
③ 라벨이 있는 작은 데이터를 처리하기
④ 모든 NLP 문제를 동시에 푸는 범용 모델 만들기

**해설**: 사전학습의 아이디어는 "인간이 언어 체험을 통해 일반적 이해를 먼저 형성한 뒤, 특정 작업으로 세분화한다"는 원리에서 비롯되었다. 모델은 문맥에서 단어를 예측하거나 마스크된 단어를 복구하면서 자동으로 의미론적, 통사론적 패턴을 학습한다.

---

### 문항 5-2. 파인튜닝(Fine-tuning)
**파인튜닝**의 개념을 가장 정확히 설명한 것은?

① 사전학습 모델의 모든 가중치를 재초기화하고 처음부터 훈련한다
② **사전학습한 모델의 가중치를 출발점으로, 목표 데이터(작은 라벨 데이터)로 마지막 몇 개 레이어(또는 전체)를 조정하여 특정 작업에 맞춘다** ✓
③ 사전학습 모델을 고정하고 새로운 레이어만 추가한다
④ 사전학습한 모델의 출력을 그대로 사용한다

**해설**: 파인튜닝은 "이미 학습한 일반적 지식을 바탕으로, 목표 작업의 고유한 특징을 빠르게 습득"하는 과정이다. 예를 들어 BERT를 감정 분류로 파인튜닝하려면, 마지막 레이어만 감정 분류용으로 업데이트하고 나머지는 거의 유지한다. 이는 작은 라벨 데이터로도 좋은 성능을 낸다.

---

### 문항 5-3. BERT의 구조
**BERT(Bidirectional Encoder Representations from Transformers)**의 특징은?

① GPT처럼 자동 회귀(왼쪽에서 오른쪽) 방식으로 훈련된다
② **양방향(양쪽 문맥을 모두 본) Transformer Encoder로, 마스크된 언어 모델(MLM) 작업으로 훈련된다** ✓
③ Transformer Decoder만 사용한다
④ 단어 순서를 무시한다

**해설**: BERT는 Transformer Encoder 여러 개를 쌓은 구조이다. 핵심은 "양방향"인데, 이는 각 단어가 문장의 좌우 모든 문맥을 동시에 본다는 의미이다. MLM (Masked Language Modeling) 작업에서, 무작위로 단어를 [MASK] 토큰으로 대체한 뒤 원래 단어를 예측하는 방식으로 훈련된다. 이를 통해 모델은 문맥을 깊이 있게 이해한다.

---

### 문항 5-4. GPT의 구조
**GPT(Generative Pre-trained Transformer)**의 특징은?

① Encoder만 사용한다
② **Decoder만 사용하는 자동 회귀(Causal) 모델로, 왼쪽에서 오른쪽으로 토큰을 순차 생성하면서 다음 토큰 예측(CLM) 작업으로 훈련된다** ✓
③ Encoder와 Decoder를 모두 사용한다
④ 양방향 주의를 사용한다

**해설**: GPT는 Transformer Decoder 여러 개로 구성되며, 자동 회귀 방식으로 작동한다 (다음 토큰 예측). 마스킹이 있어서 현재 위치 이후의 토큰을 못 본다. 이 구조는 텍스트 생성 작업에 자연스럽고, 생성 능력이 뛰어나다. ChatGPT, GPT-4 등이 GPT 아키텍처 기반이다.

---

### 문항 5-5. BERT vs GPT의 선택
다음 중 **BERT를 사용하기 적합한 작업**은?

① 글을 처음부터 끝까지 생성하기
② **주어진 문장의 감정을 분류하거나, 문장에서 특정 단어가 뜻하는 바를 이해하는 이해 작업** ✓
③ 대화를 자동으로 계속하기
④ 시나리오 전체를 창작하기

**해설**: BERT는 양방향 문맥을 모두 본 후 결정을 내리는 모델이므로, 분류(Classification), 이름 인식(NER), 질문 답변(QA) 같이 "이해"를 요구하는 작업에 적합하다. 반면 GPT는 왼쪽에서 오른쪽으로만 토큰을 생성하는 자동 회귀 모델이므로, 텍스트 완성, 창작 같은 "생성" 작업에 적합하다.

---

### 문항 5-6. Transfer Learning의 이점
**Transfer Learning (전이 학습)**의 주요 이점은?

① 모든 모델의 성능을 100% 높인다
② **대규모 데이터로 사전학습한 일반적 지식을 활용하여, 작은 라벨 데이터만으로 목표 작업의 성능을 빠르게 달성한다** ✓
③ 더 깊은 신경망을 의미한다
④ 계산 비용을 증가시킨다

**해설**: Transfer Learning은 한정된 자원(라벨 데이터, 계산력)에서도 강력한 모델을 만드는 핵심 기법이다. 예를 들어 ImageNet으로 사전학습한 CNN을 의료 이미지 분류로 파인튜닝하면, 처음부터 훈련하는 것보다 훨씬 빠르고 좋은 성능을 낸다.

---

### 문항 5-7. 마스크된 언어 모델(MLM) vs 다음 토큰 예측(CLM)
다음 중 **MLM과 CLM의 차이**를 정확히 설명한 것은?

① MLM과 CLM은 동일한 작업이다
② **MLM은 양방향 문맥을 보고 마스크된 토큰을 예측하는 작업(BERT)이고, CLM은 왼쪽 토큰만 보고 다음 토큰을 예측하는 작업(GPT)이다** ✓
③ MLM은 더 효율적이고 CLM은 비효율적이다
④ 두 작업 모두 우뇌와 좌뇌를 동시에 사용한다

**해설**: MLM (Masked Language Modeling)은 "나 [MASK] 개가 좋다"에서 [MASK] 자리에 오는 단어를 예측하는 작업이다. 양방향 문맥 "나"와 "개가 좋다"를 모두 본다. CLM (Causal Language Modeling)은 "나는"을 보고 "개를" 예측하는 방식으로, 왼쪽 문맥만 사용한다. 이 차이는 아키텍처와 용도를 결정한다.

### 문항 5-8. 신경망 기반 모델의 발전
**NLP 모델의 진화 과정**을 순서대로 나열한 것은?

① Word2Vec → MLP → RNN → LSTM → Transformer → BERT/GPT
② Transformer → LSTM → RNN → Word2Vec → MLP → BERT/GPT
③ **RNN → LSTM → Transformer → BERT/GPT (각 단계에서 특정 문제를 해결하며 발전)** ✓
④ MLP → GPT → BERT → Transformer → LSTM → RNN

**해설**: NLP 모델 발전의 흐름은 (1) RNN: 순차 처리 (2) LSTM: 장기 의존성 해결 (3) Transformer: 병렬 처리 + 모든 위치 간 상호작용 (4) BERT/GPT: 양방향 또는 생성형 사전학습이다. 각 단계는 이전 단계의 한계를 해결하면서 등장했다.

이 코드에서 **"system" 역할**의 목적은?

③ 사용자 입력을 검증한다
④ API 비용을 계산한다

**해설**: ChatGPT의 메시지는 시스템-사용자-어시스턴트의 역할 계층으로 구성된다. "system" 역할은 모델의 동작을 안내하는 지침으로, "당신은 한국사 선생님"처럼 역할 설정이나 일반 규칙을 정의한다. 이를 통해 모델의 응답 스타일과 품질을 조절할 수 있다.

### 문항 6-2. 프롬프트 엔지니어링
다음 두 프롬프트 중 **더 좋은 응답을 유도할 가능성이 높은 것**은?

프롬프트 A: "파이썬에 대해 설명해줘"
프롬프트 B: "초급자도 이해할 수 있도록, 파이썬이 무엇인지, 왜 배워야 하는지, 간단한 예제 코드와 함께 설명해줘"

① 프롬프트 A가 더 좋다 (간결함)
② **프롬프트 B가 더 좋다 (구체적 요구사항, 대상 청자, 포맷 명시)** ✓
③ 두 프롬프트는 동일한 결과를 낸다
④ 프롬프트 길이가 모델 성능과 무관하다

**해설**: 프롬프트 엔지니어링의 핵심은 **구체성**이다. "설명해줘"라는 단순한 요청보다, "초급자 대상", "예제 포함", "왜"라는 구체적 요소를 지정하면 모델은 더 정확히 의도를 파악한다. 이는 일반인에게 "뭐 해?"라고 물을 때와 "오후 3시 회의에서 분석 자료 준비해줄 수 있니?"라고 물을 때의 차이와 같다.

### 문항 6-3. 온도(Temperature) 파라미터
**온도(Temperature) 파라미터**를 설명한 것 중 맞는 것은?

① 모델의 성능을 높이는 파라미터이다
② **다음 토큰 예측 시의 무작위성을 조절하는 파라미터로, 높으면 창의적(다양한) 응답, 낮으면 일관적(집중된) 응답이 나온다** ✓
③ API 호출 비용을 결정한다
④ 항상 1.0으로 고정해야 한다

**해설**: 온도는 softmax의 "열"을 제어한다. 낮은 온도(예: 0.2)는 softmax를 "차갑게" 만들어, 가장 확률 높은 토큰에 대부분의 확률을 집중시킨다 (결정적). 높은 온도(예: 0.9)는 softmax를 "뜨겁게" 만들어, 확률을 균등하게 분산시킨다 (다양성). 사실 정보(예: 데이터 분석)에는 낮은 온도, 창작(예: 글쓰기)에는 높은 온도를 사용한다.

### 문항 6-4. Few-Shot Prompting
다음 기법의 이름과 설명을 짝지으시오.

**Few-Shot Prompting**: 모델에게 원하는 형식의 예제 2~3개를 먼저 보여준 뒤, 새로운 입력에 동일한 형식으로 응답하도록 유도하는 기법

이 기법의 이점은?

① 모델이 충분히 똑똑하면 예제가 필요 없다
② **프롬프트만 주는 것보다, 예제 패턴을 보여줌으로써 모델이 올바른 형식과 스타일을 더 정확히 따르게 한다** ✓
③ API 호출 비용을 절약한다
④ 모든 경우에 적용 가능하다

**해설**: Few-Shot Prompting은 인간 학습과 유사하다. "보고서를 작성해줘"보다 "다음처럼 형식을 맞춰서 작성해줘 [예제 3개]"라고 하면 모델이 패턴을 빠르게 이해한다. 이는 특히 구조화된 출력(JSON, CSV 등)이 필요할 때 효과적이다.
