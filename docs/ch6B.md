### 바이브코딩 활용 원칙

이번 실습은 완성된 코드를 그대로 실행하는 데서 끝나지 않는다. 학생은 Copilot, ChatGPT, Claude Code 같은 AI 보조 도구를 사용하여 코드를 읽고, 실행 오류를 해결하고, 결과 해석 표를 작성한다. 이를 바이브코딩 방식으로 운영한다.

바이브코딩은 "AI에게 전부 맡기는 방식"이 아니다. 학생이 목표를 말하고, AI가 제안한 코드를 실행하며, 출력이 맞는지 사람이 확인하는 협업 방식이다. 따라서 결과를 제출할 때는 AI가 만든 코드보다 학생이 어떤 결과를 확인했고 어떻게 해석했는지가 더 중요하다.

바이브코딩 기본 절차는 다음과 같다.

```text
1. 목표를 한 문장으로 말한다.
2. AI에게 필요한 코드 수정이나 실행 방법을 요청한다.
3. 제안된 코드를 실행한다.
4. 오류 메시지와 출력 결과를 확인한다.
5. 결과가 맞는지 사람이 해석한다.
```

이번 실습에서 사용할 수 있는 요청 예시는 다음과 같다.

| 상황 | 바이브코딩 요청 예시 |
|---|---|
| 코드 구조 파악 | "`6-2-bertopic-basic.py`의 함수별 역할을 초보자용으로 설명해줘." |
| 실행 오류 해결 | "이 BERTopic 실행 오류의 원인을 설명하고, 수정할 부분만 알려줘." |
| 결과 해석 | "`get_topic_info()` 출력에서 토픽 수와 문서 수를 어떻게 해석해야 하는지 알려줘." |
| 토픽 이름 붙이기 | "다음 키워드 목록을 보고 토픽 이름 후보 3개를 제안해줘. 단, 내가 최종 선택할 수 있게 이유도 써줘." |

주의할 점도 있다. AI가 제안한 토픽 이름은 정답이 아니라 후보이다. 최종 토픽 이름은 키워드와 대표 문서를 함께 확인한 뒤 사람이 결정한다.

## 6.1 BERTopic 개념 정리

### 토픽 모델링이란?

토픽 모델링은 여러 문서에서 반복적으로 등장하는 주제를 자동으로 찾는 방법이다. 예를 들어 고객 리뷰 1,000개를 분석하면 "배송", "품질", "가격", "환불" 같은 주제가 발견될 수 있다.

토픽 모델링의 결과는 보통 세 가지로 확인한다.

| 결과 | 의미 |
|---|---|
| 토픽 번호 | 모델이 찾은 주제의 ID |
| 대표 키워드 | 해당 토픽을 설명하는 주요 단어 |
| 문서 수 | 그 토픽에 속한 문서 개수 |

### BERTopic의 기본 흐름

BERTopic은 문서를 임베딩으로 바꾼 뒤, 의미가 비슷한 문서끼리 묶고, 각 묶음을 대표하는 키워드를 추출한다. 이 회차에서는 UMAP, HDBSCAN, c-TF-IDF의 세부 원리보다 결과를 읽고 해석하는 방법을 중심으로 실습한다.

```text
문서 데이터
→ 문서 임베딩
→ 비슷한 문서끼리 군집화
→ 토픽별 대표 키워드 추출
→ 사람이 토픽 이름 해석
```

좋은 토픽은 대표 키워드가 서로 잘 어울리고, 그 토픽에 속한 문서들도 비슷한 내용을 말한다.

## 6.4 실습 2: BERTopic 기본 실행

### 목표

BERTopic 모델을 만들고 문서에서 토픽을 추출한다.

코드의 핵심 흐름은 다음과 같다.

```python
topic_model = create_bertopic_model()
topics, probs, topic_info = train_bertopic(texts, topic_model)
```

`create_bertopic_model()`은 BERTopic 모델을 만든다. 이 함수 안에는 임베딩 모델, 차원 축소, 군집화, 단어 벡터화 설정이 포함되어 있다. 오늘은 각 파라미터의 세부 의미를 외우지 않는다.

`train_bertopic()`은 문서를 모델에 넣어 토픽을 찾는다.

```python
topics, probs = topic_model.fit_transform(texts)
```

### 바이브코딩 과제

AI 보조 도구에 다음처럼 요청한다.

```text
BERTopic 모델 생성 코드에서 embedding_model, umap_model, hdbscan_model, vectorizer_model의 역할을 각각 한 문장으로 설명해줘.
파라미터 튜닝 설명은 빼고, 이번 실습에서 결과 해석에 필요한 수준으로만 설명해줘.
```

그 다음 실제 코드를 실행한다. 실행 중 오류가 발생하면 오류 메시지 전체를 AI에게 붙여 넣고, "수정해야 할 부분만" 요청한다. 패키지 설치, 모델 다운로드, 데이터 다운로드 문제를 구분해서 확인한다.

여기서 `topics`는 각 문서가 어느 토픽에 속하는지를 나타내는 번호 목록이다. 예를 들어 `topics[0] == 2`라면 첫 번째 문서는 2번 토픽에 속한다는 뜻이다.

### 실행 중 확인할 출력

실행 중 다음 항목을 확인한다.

| 출력 | 해석 |
|---|---|
| 발견된 토픽 수 | 모델이 자동으로 찾은 주제 개수 |
| 노이즈 문서 수 | 어느 토픽에도 잘 속하지 않은 문서 수 |
| 축소 후 토픽 수 | 해석하기 쉽게 줄인 토픽 수 |

`-1` 토픽은 노이즈 문서이다. 이는 모델이 특정 토픽으로 분류하기 어렵다고 판단한 문서이다.

## 6.5 실습 3: 토픽 결과 해석

### 목표

토픽 번호와 키워드만 보고 끝내지 않고, 사람이 이해할 수 있는 이름을 붙인다.

토픽 요약은 다음 코드로 확인한다.

```python
topic_info = topic_model.get_topic_info()
print(topic_info.head())
```

특정 토픽의 대표 키워드는 다음 코드로 확인한다.

```python
topic_model.get_topic(0)
```

결과는 보통 다음과 같은 형태이다.

```text
[('game', 0.04), ('team', 0.03), ('player', 0.03), ...]
```

왼쪽은 단어이고, 오른쪽 숫자는 해당 토픽에서 그 단어가 얼마나 대표적인지를 나타내는 점수이다. 초보자 단계에서는 숫자를 자세히 해석하기보다 상위 단어들이 어떤 주제로 묶이는지 보는 것이 중요하다.

### 바이브코딩 과제

각 토픽의 대표 키워드 5~10개를 AI에게 전달하고, 토픽 이름 후보를 받는다.

```text
다음 BERTopic 키워드를 보고 토픽 이름 후보 3개를 제안해줘.
각 후보는 5단어 이내로 작성하고, 왜 그렇게 붙였는지 한 문장으로 설명해줘.

키워드:
game, team, player, season, baseball
```

AI가 제안한 이름 중 하나를 그대로 쓰지 말고, 대표 문서까지 확인한 뒤 최종 이름을 직접 정한다.

### 토픽 이름 붙이기

다음 표를 채운다.

| 토픽 ID | 대표 키워드 5개 | 문서 수 | 내가 붙인 토픽 이름 | 이유 |
|---:|---|---:|---|---|
| 0 |  |  |  |  |
| 1 |  |  |  |  |
| 2 |  |  |  |  |
| 3 |  |  |  |  |

좋은 토픽 이름은 구체적이다.

| 좋지 않은 이름 | 좋은 이름 |
|---|---|
| 문제 | 의료 질병 논의 |
| 컴퓨터 | 컴퓨터 그래픽 파일 |
| 스포츠 | 야구 경기와 선수 |
| 정치 | 정부와 정책 논쟁 |

대표 키워드만 보고 판단하기 어렵다면 해당 토픽에 속한 문서 일부를 읽어본다. 키워드와 대표 문서가 함께 맞아야 좋은 해석이다.

## 6.7 실습 5: 동적 토픽 모델링

### 동적 토픽 모델링이란?

기본 BERTopic 분석은 전체 문서를 한 번에 묶어 "어떤 주제가 있는가"를 보여준다. 그러나 뉴스, 민원, 리뷰, SNS 글처럼 날짜가 있는 문서에서는 "그 주제가 언제 늘고 줄었는가"도 중요하다.

동적 토픽 모델링은 문서에 시간 정보를 붙여 토픽의 변화 흐름을 보는 방법이다. 예를 들어 코로나 뉴스 데이터에서는 2020년 초에 확진과 방역 토픽이 늘고, 이후 백신과 재택근무 토픽이 등장할 수 있다.

### 실행 파일

동적 토픽 모델링은 다음 파일로 도입한다.

```text
practice/chapter6/code/6-3-bertopic-dynamic.py
```

이 코드는 한국어 뉴스 시뮬레이션 데이터를 사용한다. 문서마다 날짜가 있으므로, 시간에 따라 어떤 토픽이 늘거나 줄었는지 볼 수 있다. 최초 실행 시 데이터 파일이 없으면 코드가 간단한 뉴스 데이터를 생성한다.

핵심 코드는 다음 흐름이다.

```python
topics_over_time = topic_model.topics_over_time(
    texts,
    timestamps,
    nr_bins=12
)
```

### 바이브코딩 과제

동적 토픽 모델링 코드를 실행하기 전에 AI에게 다음을 요청한다.

```text
`6-3-bertopic-dynamic.py`에서 load_data(), topics_over_time(), visualize_topic_lifecycle()이 각각 어떤 역할을 하는지 설명해줘.
코드를 새로 만들지 말고, 기존 코드의 흐름을 읽는 데 필요한 설명만 해줘.
```

실행 후에는 시간별 결과를 보고 다음 요청을 사용할 수 있다.

```text
다음은 시간별 토픽 분석 결과의 일부이다.
증가한 토픽, 급증한 토픽, 비교적 안정적인 토픽을 구분해서 해석해줘.
단, 최종 해석은 내가 수정할 수 있도록 근거를 함께 제시해줘.
```

### 결과 해석

동적 토픽 분석에서는 복잡한 생명주기 계산보다 먼저 다음 세 가지를 확인한다.

1. 시간이 지나며 증가한 토픽은 무엇인가?

3. 시간이 지나며 감소한 토픽은 무엇인가?

## 6.8 LLM 토픽 레이블링 소개

BERTopic의 기본 결과는 토픽 이름이 아니라 키워드 목록이다. 예를 들어 다음 키워드가 나왔다고 하자.

```text
배송, 택배, 지연, 도착, 늦음
```

사람은 이를 보고 "배송 지연 불만"이라고 이름을 붙일 수 있다. LLM 토픽 레이블링은 이 이름 붙이기 과정을 LLM에게 맡기는 방법이다.

예시 프롬프트는 다음과 같다.

```text
다음 키워드와 대표 문서를 보고 토픽 이름을 5단어 이내로 붙여라.

키워드:
배송, 택배, 지연, 도착, 늦음

대표 문서:
배송이 늦어서 행사 날짜에 맞추지 못했다.

출력 형식:
토픽 이름:
이유:
```

LLM 레이블링은 많은 토픽에 빠르게 이름을 붙일 수 있다는 장점이 있다. 그러나 API Key와 비용이 필요하고, 생성된 이름이 항상 맞는 것은 아니다. 따라서 이번 실습에서는 개념만 소개하고, 실제 토픽 이름은 학생이 직접 붙인다.

참고 코드는 다음 위치에 있다.

```text
../dataScience/practice/chapter5/code/5-4-llm-labeling.py
```
